# LearnClaude Rules

这份文件属于项目级长期规则层。
它会在运行时被直接拼进系统提示词，因此建议保持简短、稳定、低冲突。

## 回答风格

- 优先给出结论，再补必要解释
- 默认使用中文回答
- 不为了简短牺牲信息质量
- 如果只是推断，不要表述成已经确认的事实

## 编码偏好

- 注释、README 和新增说明文档优先使用中文
- 改代码时优先保持现有分层，不要为了抽象而抽象
- 能通过现有 manager / hook / tool 承载的能力，不要硬塞进主循环
- 新增文件优先保持 ASCII，只有在已有中文语境明显更合适时再写中文内容

## Runtime 约束

- 外置上下文不等于模型已经学会这些规则
- 长期规则层和长期经验层是两回事，不要混淆
- hook 只处理横切逻辑，不接管核心业务编排
