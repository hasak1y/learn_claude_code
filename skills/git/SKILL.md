# Git Skill

在处理 Git 相关任务时，优先遵循下面的方法：

1. 先查看当前状态
- 使用 `git status --short` 或 `git status`
- 先确认工作区是否已有用户未提交的改动

2. 再查看具体差异
- 使用 `git diff`
- 如果要看已暂存改动，使用 `git diff --cached`

3. 不要随意回滚用户改动
- 不要默认执行 `git reset --hard`
- 不要默认执行 `git checkout -- <file>`
- 如果存在不属于当前任务的改动，应避免覆盖

4. 提交前做最小核对
- 确认修改范围和任务一致
- 确认没有把敏感信息写进仓库
- 如有测试，先运行最小必要验证

5. 输出时要说明
- 改了什么
- 有没有验证
- 有没有未处理风险
