# Test Skill

在处理测试、验证或排查失败时，优先遵循下面的方法：

1. 先找最小验证路径
- 优先跑与当前改动直接相关的最小测试
- 如果没有测试框架，至少做一次可重复的最小运行验证

2. 控制验证成本
- 不要一上来跑最重的全量测试
- 只有在局部验证不足以建立信心时，再扩大范围

3. 失败后先定位层次
- 是语法错误
- 是导入/配置错误
- 是运行时逻辑错误
- 还是外部依赖或网络问题

4. 记录验证结果
- 说明跑了什么
- 说明是否通过
- 如果没跑，说明原因

5. 面向当前任务收敛
- 只关注和当前改动强相关的失败
- 不要把无关的旧问题混进当前结论
